    What this actually is: a deterministic rule engine reading your own stocks, decay rates, and event history — the same kind of logic behind the dashboard. It is not a large language model reasoning over your documents. A true "reads your Hansard, contracts, and correspondence" advisor would need an LLM API connected to a small backend, which has a modest ongoing cost (a live Claude API connection) rather than the zero-cost target here. Ask if you want that wired in as a v2 — it's a different build, not a bigger version of this file.
